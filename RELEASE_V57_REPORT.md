# تقرير إصدار خدماتي v57

## الهدف

معالجة الملاحظات المرئية في رحلة الطلب المباشر، تسجيل المزود، بطاقات البحث، تحرير الصور وإشعارات الإدارة، مع إبقاء بنية التطبيق وبياناته ومساراته الحالية دون إعادة بناء.

## التنفيذ

- عرض الأقسام والخدمات في ثلاث بطاقات لكل صف من عرض 320 بكسل.
- الانتقال تلقائياً إلى الموقع بعد اختيار خدمة متاحة، مع شريط يلخص القسم والخدمة وزر رجوع يحفظ الاختيار.
- جعل نافذة الطلب بطول الهاتف وبتمرير واحد، وإلغاء أي زر ثابت يغطي البطاقات.
- إضافة خلية المزودين المطابقين إلى مراجعة الطلب لإكمال شبكة 2×2.
- إضافة تاريخ انتهاء الرخصة أو السجل إلى تسجيل المزود والتحقق منه في الواجهة والخادم.
- حماية توافق طلبات التسجيل القديمة عبر تطبيق شرط التاريخ على تسجيلات v57 الجديدة فقط عند اعتماد الإدارة.
- تحديث محرر الصور بالسحب واللمس والتكبير والتصغير والتدوير، وإضافة تعديل وحذف لكل معاينة.
- إعادة تركيب بطاقة المزود على الهاتف بصورة كاملة وتدرج شفاف للمعلومات، مع أزرار تفاصيل قصيرة ومتوازنة.
- إصلاح إجراءات إشعارات الإدارة ومنع خروجها أو اختفاء زر الحذف.
- تحديث Cache App Shell إلى v57 وإضافة طبقة التصميم الجديدة.

## التحقق

- فحص Python وترجمة الملفات.
- اختبارات النطاق.
- اختبار API مع قاعدة ورفع ملفات معزولين.
- تدقيق أمني للـ API.
- رحلة واجهة كاملة بالعربية والإنجليزية للمستخدم والمزود والإدارة على 320×700 و390×844، مع فحص Console والتجاوز الأفقي.
- فحص أداء الإطلاق ومطابقة ملفات الجذر مع `public`.

## التوافق

لم تتغير مفاتيح LocalStorage أو بنية الحسابات الحالية. الحقول الجديدة لها قيم فارغة افتراضية عند قراءة البيانات القديمة، وطلبات التسجيل القديمة لا تُرفض لمجرد أنها سبقت حقل تاريخ الانتهاء.
