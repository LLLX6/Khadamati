# تقرير إصدار خدماتي v56

تاريخ التحقق: 22 يوليو 2026

## الهدف

إزالة الرموز العامة المتكررة من الأقسام والخدمات، إصلاح الوصول إلى متابعة الطلب المباشر على الهاتف، وتحويل تقارير الإدارة إلى مركز تشغيلي قابل للتصدير بالعربية والإنجليزية.

## ما تم تنفيذه

- إضافة مكتبة رسوم SVG دلالية خفيفة تشمل الأقسام والخدمات الحالية، بألوان ثنائية متوافقة مع الوضعين الفاتح والداكن.
- ربط الرسم بهوية القسم أو الخدمة عبر `iconKey` مع قيمة افتراضية آمنة للبيانات القديمة.
- استخدام الرسوم نفسها في الرئيسية والخدمات والبحث والطلب وساحة الطلبات ولوحة إدارة الأقسام.
- إضافة منتقي رسوم بصري داخل إنشاء القسم والخدمة وتعديلهما، مع معاينة فورية.
- إبقاء أيقونات الإجراءات المألوفة مثل الرجوع والبحث والطباعة مستقلة وواضحة، حتى لا تختلط صورة الخدمة بوظيفة الزر.
- جعل جسم نافذة الطلب المباشر قابلاً للتمرير داخل الشاشة وتثبيت زر المتابعة فوق مساحة الأمان في iPhone.
- تمرير زر المتابعة إلى موضع ظاهر فور اختيار الخدمة من شاشة صغيرة.
- إعادة بناء تبويب التقارير على مؤشرات التشغيل والتحويل والجمهور والاشتراكات والمال والجودة.
- إضافة تصدير CSV بترميز UTF-8 BOM، وDOCX فعلي، وصفحة طباعة متجاوبة يمكن حفظها PDF.
- توحيد لغة التقرير واتجاهه حسب لغة التطبيق، ومنع تضمين كلمات المرور والوثائق والمواقع الدقيقة.
- تحديث App Shell ونسخة الذاكرة المؤقتة إلى v56.

## اختبارات الإصدار

- فحص Python وJavaScript.
- 14 اختبار نطاق واشتراكات ومطابقة وخصوصية.
- اختبار API معزول يشمل التسجيل والموافقة والطلب والمحادثة والتوصية والدفع والموافقة على التواصل.
- التحقق من CSV العربي وDOCX الإنجليزي وصفحة الطباعة LTR.
- اختبار أمان API والجلسات والوسائط وCORS ومنع IDOR والكاش الخاص.
- رحلة واجهة كاملة على 390x844 و320x700 للمستخدم والمزود والإدارة.
- تحقق بصري من تنوع الرسوم وعدم خروج البطاقات أو نافذة الطلب من الشاشة.
- اختبار أداء الإقلاع المحلي وبلوغ الواجهة حالة التفاعل ضمن الهدف المعتمد.

## التوافق

لا توجد Migration مدمرة. حقل `iconKey` اختياري، ويُستكمل تلقائياً عند قراءة بيانات قديمة. تظل الحقول القديمة `icon` محفوظة للتوافق، لكن الواجهة الحديثة لا تعتمد عليها لرسم الأقسام والخدمات.
